# Code of Conduct

## Our Commitment

BFlow is committed to providing a respectful, collaborative, and professional environment for everyone contributing to the project.

The goal of this project is to build reliable financial software while maintaining healthy and constructive communication between contributors.

## Expected Behavior

Contributors are expected to:

- Be respectful and professional in discussions
- Accept constructive feedback gracefully
- Focus discussions on improving the project
- Help maintain a welcoming environment for new contributors
- Respect different levels of experience and technical backgrounds

## Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment or personal attacks
- Discrimination of any kind
- Toxic, hostile, or intentionally disruptive behavior
- Public or private intimidation
- Sharing confidential or sensitive information without permission
- Spam or intentionally harmful contributions

## Contribution Standards

When contributing to BFlow:

- Keep pull requests focused and technically clear
- Follow existing project conventions
- Avoid unnecessary architectural complexity
- Prioritize security, readability, and maintainability
- Test changes before submitting

## Project Philosophy

BFlow values:

- Simplicity over unnecessary complexity
- Correctness over speed
- Secure design over shortcuts
- Real-world usability over feature bloat

## Enforcement

Project maintainers reserve the right to remove, reject, or revert contributions and interactions that violate this Code of Conduct.

Repeated or severe violations may result in temporary or permanent restriction from contributing to the project.

## Contact

For concerns regarding conduct or contributions, contact the project maintainer directly through GitHub.
